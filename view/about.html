MPower App

The MPower Application




MPower is an adaptive power management system for Android based devices. An adaptive power management system is a system able to exploit the Observe-Decide-Act loop to adapt to the current environmental situation and the specific device it has to optimize. In this process, also the user experience cannot be disregarded. The final outputs of our system will be user-specific power consumption profiles. The system will be able to optimize the device power consumption based on the device characteristics, the environmental conditions and the user need. Those components have to be considered at the same time. To reach this final goal, it is necessary to model the device power consumption on a component base, e.g, the wifi interface power consumption given its status and the amount of data that it is transferring. Once the model has been generated automatically for each type of device, it is possible to add the user experience to tune the previous generated model to adapt to the user specific needs. The feedback provided by the user is reported to the adaptive system that will tune its decisions accordingly. To automatically build device-specific power consumption models, it is necessary to have a large dataset of data to work on.





System Architecture


One of the fundamental MPower features is the automatic generation of power models. The model generator is the core component in the system. It is used to evaluate the power consumption of the system in a certain state, it is constantly refined using the information retrieved from the user habits and the experience collected on the device and it is personalized for each user. Since it is impossible to retrieve data about the device power consumption in a static way (e.g., using a data acquisition instrument) due to the great amount of different de- vices running Android, it is necessary to generate completely customized models using real data.

As a consequence, the power consumption model is automatically built using the data collected from each different device, that are analyzed on an external server. In order to retrieve all these data, the first MPower version is focalized on data collection, to retrieve as many data as possible from different devices and users. This particular aspect, the data collection phase, is described in details in Section IV. When a new device registers into MPower, it will be equipped with a generic model. The model will be then refined using the data collected from the specific device. Models are built and refined on the external server that stores the previously retrieved data, but sent to the correct device to be used online. Knowing the model generated by the remote server, on the device it is now possible to use it to know the current system power consumption and its battery life expectation starting from the current conditions.

Frequently Asked Questions

If you need information about MPower, please read this section before contacting us. Than, if you don't find what are you looking for, do not esitate! Send us an e-mail!

Registration

Why do I need to login using Google? Will my password be saved?

You need to register your phone to enable a secure data sending between your phone and our server. Your password will not be saved, since the login/registration is performed by Google.

What personal data are saved?

We are saving just your email, your first name and your phone IMEI (the phone identifier) for identification purposes.

Pricing

Is MPower a free app? Are you thinking to sell it in the future?

MPower is a university project lead by the NECSTLab@POLIMI. For this reason it is and will be a free app.

Supported devices

Is MPower available also for iOS, BlackBarry or Windows Phone?

Nope. MPower started as (and still is) a university project. Android is the only choice if you are looking for an open source system that enable you to make all the experiments you need. Moreover, the other OS doesn't allow us to have all the intruments we neee like, for instance, backgound execution of processes.

Are you planning to port MPower on other platform in the future?

We're sorry but we do not have any plan in this direction right now.

Data collection

What data are collected?

We are collecting data about your phone internal status. We'll NEVER collect any personal data, browser history, gps positions, cookies, or any other sort of tracking data.

The application is asking for a lot of permissions, why?

We need a lot of permissions to read your phone internal status, but we are not modifying any behavior without your consent.

Where are my data stored?

Daily data are stored on your SD card and are then compressed and encrypted before the sending, thous ensuring the security of the overall system.

Data sending

Do I need to send data manually?

It is not necessary, the app daily sends all stored data automatically, if a data connection is available.

What is the purpose of the button "Send data now"?

You can click on the send data now button if you want to force a data sending immediately. This was done mainly for debug purposes and will probably be changed on later versions.

When are the data sent?

Data are sent daily, whenever it is available a wifi connection.

I rarely connect to wifi, will my data get lost?

No at all, after 2 days without a wifi connection available, data are sent over the mobile network (2G/3G). If you don't want to use your mobile connection, you can disable this feature through the application preferences.

I have pressed the "Send data now" button, but a message said "No data to send", why?

It is only possible to send data related to previous days, not about the current day. If the message "No data to send" appears, it means that the phone has automatically sent your previous data.

Security

Are my data encrypted before they are sent?

Yes, they are encrypted using an asymmetrical cryptographic algorithm to ensure that no data was being manipulated.

Suggestions

What type of suggestions can MPower propose?

MPower issues a notification if wifi, or bluetooth or GPS is enabled but not in use.

Can I disable suggestions?

Yes, in the application preferences it is possible to disable each type of suggestion separately.

Can I change the delay before a notification is shown?

Sure, just enter the application preferences and change the delay time for each type of notification.

Web site

I have just sent my data, but they are not visible on the website, why?

It is not an immediate process, data are elaborated at fixed times during the day. If you happen to can't view your data even after several hours after the sending, please let us know (you need to be logged in before reporting an issue).

On the website, can I see all the data that are I have sent?

Yes, all your data are visible on the website, and only you are entitled to view them.
